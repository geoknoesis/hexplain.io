The Hexplain Audio Vocabulary (ADV) is a domain-specific ontology that provides a standard set of terms for describing the semantic properties of audio data. As a component of the Hexplain Ontology Family, it is designed to be used with the BDDO and Hexplain Core ontologies. ADV enables format authors to map low-level physical data fields from audio formats (like WAV or MP3 with ID3 tags) to high-level, unambiguous concepts (e.g., sample rate, artist, album), facilitating interoperability and semantic querying across different audio file formats.

This document is an unofficial draft produced for the purpose of defining the Hexplain standard. It has no official standing and is not endorsed by any standards body.

Introduction

Audio files contain both technical data (the encoded audio stream) and descriptive metadata (tags describing the content). While BDDO can describe the physical structure of these files, it cannot describe the meaning of fields like "sample rate" or "artist." The Audio Vocabulary provides this essential semantic layer.

By providing a common, stable set of URIs for concepts like "bit depth," "album," and "track number," the ADV allows a Hexplain processor to understand and compare audio content from different file formats. This enables high-level queries such as "Find all audio files by the artist 'The Beatles' with a sample rate of 44100 Hz."

Conformance

The key words "MUST", "MUST NOT", "REQUIRED", "SHALL", "SHALL NOT", "SHOULD", "SHOULD NOT", "RECOMMENDED", "MAY", and "OPTIONAL" in this document are to be interpreted as described in [[RFC2119]].

Core Concepts (Classes)

ADV defines a set of classes to represent the high-level concepts within an audio file.

adv:AudioContainer
The core class representing the audio file as a whole (e.g., a WAV or MP3 file).
adv:AudioStream
Represents the block containing the actual encoded audio sample data.
adv:MetadataTag
Represents a metadata block (like an ID3 or Vorbis comment tag) that contains descriptive information about the audio content.

Semantic Properties

These properties describe the attributes of audio streams and metadata tags. In a Hexplain description, a bddo:Field would use the hexplain:mapsToProperty predicate to link to one of these semantic properties.

Property Description Expected Range
adv:sampleRate The number of samples per second in Hertz (e.g., 44100, 48000). xsd:positiveInteger
adv:audioChannels The number of audio channels (e.g., 1 for mono, 2 for stereo). xsd:positiveInteger
adv:bitDepth The number of bits per audio sample (e.g., 16, 24). Also known as sample format. xsd:positiveInteger
adv:duration The duration of the audio in seconds. xsd:decimal
adv:bitrate The average bitrate of the track in bits per second. xsd:positiveInteger
adv:codecIdentifier A string identifying the audio codec (e.g., "PCM", "MP3", "FLAC"). xsd:string
adv:artist The name of the recording artist. xsd:string
adv:album The name of the album the audio belongs to. xsd:string
adv:trackTitle The title of the audio track. xsd:string
adv:trackNumber The number of the track on its original album. xsd:positiveInteger
adv:genre The genre of the audio content. xsd:string

Layout Axes

ADV extends the Hexplain Data Layout Vocabulary (DLV) by defining standard axes specific to audio. These are used in a dlv:Dimension to specify what each dimension of a data array represents.

adv:axisSample
Represents the sample axis in a sequence of audio samples (often corresponds to time).
adv:axisChannel
Represents the channel axis for interleaved audio (e.g., Left, Right channels).

Normative OWL Ontology

The following is the complete, normative OWL ontology for ADV 1.0, expressed in Turtle syntax [[turtle]].

@prefix : <https://hexplain.io/ns/audio#> .
@prefix owl: <http://www.w3.org/2002/07/owl#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix dlv: <https://hexplain.io/ns/dlv#> .

<https://hexplain.io/ns/audio> a owl:Ontology ;
    owl:versionInfo "1.0" ;
    rdfs:label "Hexplain Audio Vocabulary (ADV)" .

# -- Classes --
:AudioContainer a owl:Class ; rdfs:label "Audio Container" .
:AudioStream a owl:Class ; rdfs:label "Audio Stream" .
:MetadataTag a owl:Class ; rdfs:label "Metadata Tag" .

# -- Properties --
:sampleRate a owl:DatatypeProperty ; rdfs:label "Sample Rate" .
:audioChannels a owl:DatatypeProperty ; rdfs:label "Audio Channels" .
:bitDepth a owl:DatatypeProperty ; rdfs:label "Bit Depth" .
:duration a owl:DatatypeProperty ; rdfs:label "Duration" .
:bitrate a owl:DatatypeProperty ; rdfs:label "Bitrate" .
:codecIdentifier a owl:DatatypeProperty ; rdfs:label "Codec Identifier" .
:artist a owl:DatatypeProperty ; rdfs:label "Artist" .
:album a owl:DatatypeProperty ; rdfs:label "Album" .
:trackTitle a owl:DatatypeProperty ; rdfs:label "Track Title" .
:trackNumber a owl:DatatypeProperty ; rdfs:label "Track Number" .
:genre a owl:DatatypeProperty ; rdfs:label "Genre" .

# -- Axes for DLV --
:axisSample a dlv:Axis ; rdfs:label "Sample Axis" .
:axisChannel a dlv:Axis ; rdfs:label "Channel Axis" .

Example of Usage

This example shows a fragment of a WAV file description, demonstrating how the physical fields of the format (`fmt `) chunk are mapped to the semantic properties in the Audio Vocabulary.

@prefix bddo: <https://hexplain.io/ns/bddo#> .
@prefix hexplain: <https://hexplain.io/ns/core#> .
@prefix adv: <https://hexplain.io/ns/audio#> .
@prefix wav: <https://hexplain.io/formats/wav#> .

# The 'fmt ' chunk is a physical representation of an AudioStream concept.
wav:FormatChunk a bddo:Struct ;
    hexplain:mapsToClass adv:AudioStream ;
    bddo:hasField ( wav:SampleRate, wav:Channels, wav:BitsPerSample, ... ) .

# The physical 'SampleRate' field maps to the semantic 'sampleRate' property.
wav:SampleRate a bddo:Field ;
    bddo:dataType bddo:uint32le ;
    hexplain:mapsToProperty adv:sampleRate .

# The physical 'Channels' field maps to the semantic 'audioChannels' property.
wav:Channels a bddo:Field ;
    bddo:dataType bddo:uint16le ;
    hexplain:mapsToProperty adv:audioChannels .

# The physical 'BitsPerSample' field maps to the semantic 'bitDepth' property.
wav:BitsPerSample a bddo:Field ;
    bddo:dataType bddo:uint16le ;
    hexplain:mapsToProperty adv:bitDepth .
