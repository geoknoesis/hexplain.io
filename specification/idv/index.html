The Hexplain Image Vocabulary (IDV) is a domain-specific ontology that provides a standard set of terms for describing the semantic properties of raster images. It is a component of the Hexplain Ontology Family, designed to be used in conjunction with the BDDO and Hexplain Core ontologies. IDV allows format authors to map low-level physical data fields (e.g., a 4-byte integer) to high-level, unambiguous concepts (e.g., image width in pixels), enabling interoperability and semantic querying across different image file formats.

This document is an unofficial draft produced for the purpose of defining the Hexplain standard. It has no official standing and is not endorsed by any standards body.

Introduction

While the BDDO ontology describes the physical structure of a file, it does not provide any information about what the data represents. The Image Vocabulary provides this semantic layer for the specific domain of raster imagery.

By providing a common, stable set of URIs for concepts like "pixel width," "color space," and "compression type," the IDV allows a Hexplain processor to understand and compare images from different file formats (e.g., PNG, JPEG, BMP) in a standardized way. A user can ask a high-level question like "Find all images wider than 1024 pixels" without needing to know the specific physical location of the width field in each format.

The key words "MUST", "MUST NOT", "REQUIRED", "SHALL", "SHALL NOT", "SHOULD", "SHOULD NOT", "RECOMMENDED", "MAY", and "OPTIONAL" in this document are to be interpreted as described in [[RFC2119]].

Core Concepts (Classes)

IDV defines a set of classes to represent the high-level concepts within an image file.

img:Image
The core class representing the image as a whole.
img:ImageHeader
Represents the metadata block that contains the primary image characteristics, such as dimensions and color information.
img:ColorProfile
Represents information about the color space, gamma, etc.
img:PixelData
Represents the block containing the actual compressed or uncompressed pixel data.

Semantic Properties

These properties describe the attributes of an image. In a Hexplain description, a bddo:Field would use the hexplain:mapsToProperty predicate to link to one of these semantic properties.

Property Description Expected Range
img:pixelWidth The width of the image in pixels. xsd:positiveInteger
img:pixelHeight The height of the image in pixels. xsd:positiveInteger
img:bitsPerComponent The number of bits per color or sample component (e.g., 8 for standard RGB). xsd:positiveInteger
img:colorType An identifier for the color type (e.g., Grayscale, RGB, Indexed). Often an integer code. xsd:integer
img:compressionMethod An identifier for the compression method used. xsd:integer
img:filterMethod An identifier for the filter or prediction method used before compression. xsd:integer
img:interlaceMethod An identifier for the interlacing or progressive display method. xsd:integer

Layout Axes

IDV extends the Hexplain Data Layout Vocabulary (DLV) by defining standard axes specific to imagery. These are used in a dlv:Dimension to specify what each dimension of a pixel data array represents.

img:axisX
Represents the horizontal (width) spatial axis.
img:axisY
Represents the vertical (height) spatial axis.
img:axisBand
Represents the band, channel, or component axis (e.g., R, G, B components).

Normative OWL Ontology

The following is the complete, normative OWL ontology for IDV 1.0, expressed in Turtle syntax [[turtle]].

@prefix : <https://hexplain.io/ns/image#> .
@prefix owl: <http://www.w3.org/2002/07/owl#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix dlv: <https://hexplain.io/ns/dlv#> .

<https://hexplain.io/ns/image> a owl:Ontology ;
    owl:versionInfo "1.0" ;
    rdfs:label "Hexplain Image Vocabulary (IDV)" .

# -- Classes --
:Image a owl:Class ; rdfs:label "Image" .
:ImageHeader a owl:Class ; rdfs:label "Image Header" .
:ColorProfile a owl:Class ; rdfs:label "Color Profile" .
:PixelData a owl:Class ; rdfs:label "Pixel Data" .

# -- Properties --
:pixelWidth a owl:DatatypeProperty ; rdfs:label "Pixel Width" .
:pixelHeight a owl:DatatypeProperty ; rdfs:label "Pixel Height" .
:bitsPerComponent a owl:DatatypeProperty ; rdfs:label "Bits Per Component" .
:colorType a owl:DatatypeProperty ; rdfs:label "Color Type" .
:compressionMethod a owl:DatatypeProperty ; rdfs:label "Compression Method" .
:filterMethod a owl:DatatypeProperty ; rdfs:label "Filter Method" .
:interlaceMethod a owl:DatatypeProperty ; rdfs:label "Interlace Method" .

# -- Axes for DLV --
:axisX a dlv:Axis ; rdfs:label "X Axis" .
:axisY a dlv:Axis ; rdfs:label "Y Axis" .
:axisBand a dlv:Axis ; rdfs:label "Band Axis" .

Example of Usage

This example shows a fragment of a PNG description, demonstrating how the physical fields of the `IHDR` chunk are mapped to the semantic properties in the Image Vocabulary.

@prefix bddo: <https://hexplain.io/ns/bddo#> .
@prefix hexplain: <https://hexplain.io/ns/core#> .
@prefix img: <https://hexplain.io/ns/image#> .
@prefix png: <https://hexplain.io/formats/png#> .

# The IHDR chunk data structure is a physical representation of an ImageHeader concept.
png:IHDR_ChunkData a bddo:Struct ;
    hexplain:mapsToClass img:ImageHeader ;
    bddo:hasField ( png:Width, png:Height, png:BitDepth, ... ) .

# The physical 'Width' field maps to the semantic 'pixelWidth' property.
png:Width a bddo:Field ;
    bddo:dataType bddo:uint32be ;
    hexplain:mapsToProperty img:pixelWidth .

# The physical 'Height' field maps to the semantic 'pixelHeight' property.
png:Height a bddo:Field ;
    bddo:dataType bddo:uint32be ;
    hexplain:mapsToProperty img:pixelHeight .

# The physical 'BitDepth' field maps to the semantic 'bitsPerComponent' property.
png:BitDepth a bddo:Field ;
    bddo:dataType bddo:uint8 ;
    hexplain:mapsToProperty img:bitsPerComponent .
