The Hexplain Geospatial Vocabulary (GV) is a domain-specific ontology that provides a standard set of terms for describing the semantic properties of geospatial and geointelligence data, including raster, vector, and point cloud formats. As a component of the Hexplain Ontology Family, it is designed to be used with the BDDO, DLV, and Hexplain Core ontologies. GV enables format authors to map low-level physical data fields from formats like GeoTIFF, NetCDF, HDF5, or LAS to high-level, unambiguous concepts (e.g., coordinate reference system, sensor model, point cloud layout), facilitating interoperability and semantic analysis of geospatial data.

This document is an unofficial draft produced for the purpose of defining the Hexplain standard. It has no official standing and is not endorsed by any standards body. Version 1.2 expands the vocabulary's scope to include point clouds and other geointelligence concepts. Version 1.1 added the gv:pixelRegistration property.

Introduction

Geospatial data formats contain complex information describing how data values map to physical locations on Earth. While BDDO and DLV can describe the physical structure and multi-dimensional layout of this data, they cannot describe the meaning of fields like "Coordinate Reference System" or "Sensor Platform." The Geospatial Vocabulary provides this essential semantic layer.

By providing a common, stable set of URIs for concepts like "projectionName," "pointCount," and "acquisitionTime," the GV allows a Hexplain processor to understand and georeference data from different file formats in a standardized way. This enables high-level queries such as "Find all Lidar point clouds collected after a specific date that use the WGS 84 projection."

Conformance

The key words "MUST", "MUST NOT", "REQUIRED", "SHALL", "SHALL NOT", "SHOULD", "SHOULD NOT", "RECOMMENDED", "MAY", and "OPTIONAL" in this document are to be interpreted as described in [[RFC2119]].

Core Concepts (Classes)

GV defines a set of classes to represent the high-level concepts within a geospatial file.

gv:GeospatialDataset
The core class representing the geospatial dataset as a whole.
gv:RasterDataset
A subclass of gv:GeospatialDataset for raster/gridded data.
gv:VectorDataset
A subclass of gv:GeospatialDataset for vector data (points, lines, polygons).
gv:PointCloud
A subclass of gv:GeospatialDataset for point cloud data, such as Lidar.
gv:CoordinateReferenceSystem
Represents the Coordinate Reference System (CRS) information.
gv:GeoTransform
Represents the affine transformation parameters that map grid coordinates to ground coordinates.
gv:AcquisitionInfo
Represents metadata about the collection event, including the sensor and platform.
gv:SensorModel
Represents information about the sensor used for data collection.
gv:Platform
Represents information about the platform (e.g., satellite, aircraft) that carried the sensor.

Semantic Properties

These properties describe the attributes of geospatial datasets. In a Hexplain description, a bddo:Field would use the hexplain:mapsToProperty predicate to link to one of these semantic properties.

Property Description Expected Range
gv:wktString The full Coordinate Reference System definition in Well-Known Text (WKT) format. xsd:string
gv:epsgCode The numeric EPSG code for the Coordinate Reference System. xsd:positiveInteger
gv:originLongitude The longitude of the reference point of the top-left pixel/grid cell. xsd:double
gv:originLatitude The latitude of the reference point of the top-left pixel/grid cell. xsd:double
gv:pixelRegistration Specifies whether the origin coordinate refers to the pixel's corner or center. gv:PixelCorner or gv:PixelCenter
gv:scaleX The pixel/grid width or horizontal scale in ground units. xsd:double
gv:scaleY The pixel/grid height or vertical scale in ground units (often negative). xsd:double
gv:noDataValue A special value used in a raster band to indicate that a pixel has no valid data. xsd:double or xsd:integer
gv:pointCount The total number of points in a point cloud. xsd:unsignedLong
gv:pointDataLayout Links to a bddo:Struct that defines the layout of a single point record in a point cloud. bddo:Struct
gv:acquisitionTime The timestamp of the data acquisition. xsd:dateTime
gv:platformName The name or identifier of the collection platform (e.g., "Landsat-8", "N1234A"). xsd:string
gv:sensorType The type or model of the sensor (e.g., "Lidar", "Panchromatic"). xsd:string
gv:classificationLevel The classification level of the data (e.g., "UNCLASSIFIED"). xsd:string

Layout Axes

GV extends the Hexplain Data Layout Vocabulary (DLV) by defining standard axes specific to geospatial data. These are used in a dlv:Dimension to specify what each dimension of a data array represents.

gv:axisLatitude
Represents the latitudinal (typically Y) spatial axis.
gv:axisLongitude
Represents the longitudinal (typically X) spatial axis.
gv:axisElevation
Represents an elevational or vertical (Z) spatial axis.
gv:axisPoint
Represents the point axis in a point cloud.

Normative OWL Ontology

The following is the complete, normative OWL ontology for GV 1.2, expressed in Turtle syntax [[turtle]].

@prefix : <https://hexplain.io/ns/geo#> .
@prefix owl: <http://www.w3.org/2002/07/owl#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix dlv: <https://hexplain.io/ns/dlv#> .
@prefix bddo: <https://hexplain.io/ns/bddo#> .

<https://hexplain.io/ns/geo> a owl:Ontology ;
    owl:versionInfo "1.2" ;
    rdfs:label "Hexplain Geospatial Vocabulary (GV)" .

# -- Classes --
:GeospatialDataset a owl:Class ; rdfs:label "Geospatial Dataset" .
:RasterDataset a owl:Class ; rdfs:subClassOf :GeospatialDataset ; rdfs:label "Raster Dataset" .
:VectorDataset a owl:Class ; rdfs:subClassOf :GeospatialDataset ; rdfs:label "Vector Dataset" .
:PointCloud a owl:Class ; rdfs:subClassOf :GeospatialDataset ; rdfs:label "Point Cloud" .
:CoordinateReferenceSystem a owl:Class ; rdfs:label "Coordinate Reference System" .
:GeoTransform a owl:Class ; rdfs:label "GeoTransform" .
:AcquisitionInfo a owl:Class ; rdfs:label "Acquisition Info" .
:SensorModel a owl:Class ; rdfs:label "Sensor Model" .
:Platform a owl:Class ; rdfs:label "Platform" .

# -- Properties --
:wktString a owl:DatatypeProperty ; rdfs:label "WKT String" .
:epsgCode a owl:DatatypeProperty ; rdfs:label "EPSG Code" .
:originLongitude a owl:DatatypeProperty ; rdfs:label "Origin Longitude" .
:originLatitude a owl:DatatypeProperty ; rdfs:label "Origin Latitude" .
:pixelRegistration a owl:ObjectProperty ; rdfs:domain :GeoTransform ; rdfs:label "Pixel Registration" .
:scaleX a owl:DatatypeProperty ; rdfs:label "X Scale" .
:scaleY a owl:DatatypeProperty ; rdfs:label "Y Scale" .
:noDataValue a owl:DatatypeProperty ; rdfs:label "No Data Value" .
:pointCount a owl:DatatypeProperty ; rdfs:label "Point Count" .
:pointDataLayout a owl:ObjectProperty ; rdfs:domain :PointCloud ; rdfs:range bddo:Struct ; rdfs:label "Point Data Layout" .
:acquisitionTime a owl:DatatypeProperty ; rdfs:label "Acquisition Time" .
:platformName a owl:DatatypeProperty ; rdfs:label "Platform Name" .
:sensorType a owl:DatatypeProperty ; rdfs:label "Sensor Type" .
:classificationLevel a owl:DatatypeProperty ; rdfs:label "Classification Level" .

# -- Individuals for Pixel Registration --
:PixelCorner a owl:NamedIndividual ; rdfs:label "Pixel Corner Registration" .
:PixelCenter a owl:NamedIndividual ; rdfs:label "Pixel Center Registration" .

# -- Axes for DLV --
:axisLatitude a dlv:Axis ; rdfs:label "Latitude Axis" .
:axisLongitude a dlv:Axis ; rdfs:label "Longitude Axis" .
:axisElevation a dlv:Axis ; rdfs:label "Elevation Axis" .
:axisPoint a dlv:Axis ; rdfs:label "Point Axis" .

Example of Usage

This example shows a fragment of a Lidar (LAS) file description, demonstrating how the physical header fields and the point data layout are mapped to the semantic properties in the Geospatial Vocabulary.

@prefix bddo: <https://hexplain.io/ns/bddo#> .
@prefix hexplain: <https://hexplain.io/ns/core#> .
@prefix gv: <https://hexplain.io/ns/geo#> .
@prefix las: <https://hexplain.io/formats/las#> .

# The LAS file header is a physical representation of a PointCloud concept.
las:Header a bddo:Struct ;
    hexplain:mapsToClass gv:PointCloud ;
    # The layout of each point is described by this separate Struct.
    gv:pointDataLayout las:PointRecordFormat1 ;
    bddo:hasField ( las:NumberOfPointRecords, ... ) .

# The physical 'NumberOfPointRecords' field maps to the semantic 'pointCount' property.
las:NumberOfPointRecords a bddo:Field ;
    bddo:dataType bddo:uint32le ;
    hexplain:mapsToProperty gv:pointCount .

# The BDDO description for a single point record. This provides the physical layout
# that gv:pointDataLayout refers to.
las:PointRecordFormat1 a bddo:Struct ;
    bddo:endianness bddo:LittleEndian ;
    bddo:hasField ( las:X, las:Y, las:Z, las:Intensity, ... ) .

las:X a bddo:Field ; bddo:dataType bddo:int32le .
las:Y a bddo:Field ; bddo:dataType bddo:int32le .
las:Z a bddo:Field ; bddo:dataType bddo:int32le .
las:Intensity a bddo:Field ; bddo:dataType bddo:uint16le .
